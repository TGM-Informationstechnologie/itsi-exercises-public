    "WireGuard VPN einrichten und testen" - Taskdescription

    Einführung

    in VPN (Virtuelles Privates Netzwerk) baut eine verschlüsselte Verbindung über das Internet auf, die Ihre IP-Adresse verbirgt und Ihre Online-Aktivitäten schützt. Sie eignen sich zum sicheren Surfen in öffentlichen WLAN-Netzwerken, für den Schutz der Privatsphäre vor Internetanbietern und Regierungen sowie für den sicheren Zugriff auf Unternehmens- oder Universitätsnetzwerke von zu Hause oder unterwegs aus. Die Einrichtung erfolgt oft über eine spezielle VPN-App des Anbieters oder manuell in den Netzwerkeinstellungen Ihres Geräts.

    Ziele

    • Verständnis der Funktionsweise eines VPN
    • Einrichtung einer VPN-Verbindung unter Windows
    • Überprüfung der Verbindung und Analyse des Netzwerkverkehrs

    Kompetenzzuordnung

    Grundlagen der IT Sicherheit

    • Strategien zur Erhöhung der Privatsphäre im Internet entwickeln.

    Methoden und Verfahren der IT-Sicherheit

    • grundlegende Zugriffsschutzmechanismen erklären und einrichten
    • einfache Werkzeuge der Netzwerksicherheit nennen und bedienen

    Voraussetzungen

    • Windows 10 oder 11
    • Internetverbindung
    • Wireguard Download
    • Administratorrechte auf dem Gerät

    Detaillierte Aufgabenbeschreibung

    WireGuard ist ein modernes VPN-Protokoll, das sich durch einfache Konfiguration, hohe Performance und starke Sicherheit auszeichnet. In dieser Übung lernen Sie, wie man eine VPN-Verbindung mit WireGuard unter Windows einrichtet, testet und analysiert.

    Aufgabe Grundkompetenz:

    Teil 1 - Vorbereitung

    Recherchieren Sie die Grundlagen von WireGuard:

    • Architektur (Peer-to-Peer, statische Schlüssel)
    • verwendete Kryptografie
    • Vorteile gegenüber klassischen VPNs

    Teil 2 - WireGuard Server installieren

    • Installieren Sie die Software mit Administratorrechten. Es gibt 2 Möglichkeiten

      • Manuell: https://administrator.de/tutorial/merkzettel-vpn-installation-mit-wireguard-660620.html
      • Unterstütz mittels Tool: https://goneuland.de/wg-server-for-windows-wireguard-server-unter-windows-einrichten/
      • Offizieller Download: https://www.wireguard.com/install/

    • Erstellen Sie Schlüsselpaare

    • Konfigurieren Sie den Server
    • Starten Sie den Dienst
    • Öffnen Sie den Port in der Firewall

    Teil 3 - Smartphone als Client konfigurieren

    • Installieren Sie die WireGuard-App:

      • Android: Google Play Store
      • iOS: App Store

    • Erstellen Sie Client-Schlüssel:

    • Erstellen Sie die Client-Konfiguration:

      • QR-Code generieren:

    Teil 4 - Verbindung testen

    • Aktivieren Sie die Verbindung auf dem Smartphone.
    • Prüfen Sie die IP-Adresse über https://whatismyipaddress.com.
    • Testen Sie die Verbindung zum Server mit Ping oder Traceroute (z. B. ping 10.0.0.1)
    • Überprüfen Sie die Logs auf dem Server und dem Client

    Teil 5 - Netzwerkverkehr analysieren

    Nutze Wireshark, um den Unterschied zwischen unverschlüsseltem und verschlüsseltem Verkehr zu zeigen Filter: ip.addr == [VPN-Server-IP]

    Recherchiere unterschiedliche VPN Protokolle (mind. 5).

    Erstelle eine Vergleichstabelle mit folgenden Kriterien:

    • Verschlüsselung
    • Authentifizierung
    • Datenintegrität
    • Kompatibilität
    • Performance
    • Vor- und Nachteile in der Praxis

    Aufgabe Erweiterte Kompetenz:

    Teil 6 - Logging

    Sie finden hier verschiedene Methoden. Nutzen sie mindestens eine davon.

    Teil 7A - Manuelles Logging über PowerShell

    Du kannst den Status regelmäßig abfragen und in eine Datei schreiben (z.B. logging.ps1:

    # Write Log entry
$timestamp = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
$status = & "C:\Program Files\WireGuard\wireguard.exe" /dump
"$timestamp`n$status`n" | Out-File -Append "C:\Logs\wireguard_status.log"

    Hinweise zum Schreiben eines Powershell Skriptes: PowerShell Script erstellen: Schritt-für-Schritt-Anleitung - IONOS AT

    Dies kann z. B. über den Windows Task Scheduler alle 5 Minuten ausgeführt werden.

    Die Datei wireguard_status.log enthält dann eine chronologische Übersicht der Verbindungen.

    Teil 7B - Firewall-Protokollierung aktivieren

    WireGuard selbst schreibt keine Events in die Ereignisanzeige, aber du kannst:

    • Firewall-Ereignisse protokollieren (z. B. eingehende Verbindungen auf Port 51820).
    • Netzwerkverbindungsereignisse überwachen (z. B. über „Netzwerkverbindungsprotokollierung“ in Gruppenrichtlinien).

    Firewall Protokollierung

    Öffne die Windows Defender Firewall mit erweiterter Sicherheit.

    • Klicke rechts auf "Eigenschaften" für das aktive Profil (z. B. "Privat").
    • Unter "Protokollierung":
    • Protokollierung fehlgeschlagener Verbindungen: Ja
    • Protokollierung erfolgreicher Verbindungen: Ja(optional)
    • Pfad zur Protokolldatei: z. B. C:\Windows\System32\LogFiles\Firewall\pfirewall.log
    • Maximale Dateigröße: z. B. 4096 KB

    Die Datei pfirewall.log enthält nun Einträge wie:

    2025-08-28 11:45:01 DROP UDP 192.168.1.100 192.168.1.10 51820 ...

    Du kannst die Logdatei mit PowerShell filtern:

    Select-String -Path "C:\Windows\System32\LogFiles\Firewall\pfirewall.log" -Pattern "51820"

    Oder nach IP-Adresse und Port:

    Get-Content "C:\Windows\System32\LogFiles\Firewall\pfirewall.log" | Where-Object { $_ -match "192\.168\.1\.100" -and $_ -match "51820"

    Ereignisanzeige (Event Viewer)

    Die Windows-Ereignisanzeige kann Netzwerkereignisse erfassen, wenn du Audit-Regeln aktivierst:

    Aktivierung über Gruppenrichtlinien:

    • Öffne gpedit.msc
    • Gehe zu: Computerkonfiguration → Windows-Einstellungen → Sicherheitseinstellungen → Erweiterte Überwachungsrichtlinie → Überwachung von Systemdiensten

    • Aktiviere:

      • Netzwerkverbindungsüberwachung
      • Anmeldeversuche
      • Objektzugriffe

    Du findest relevante Logs unter:

    • Event Viewer → Windows-Protokolle → Sicherheit
    • Event Viewer → Anwendungen und Dienste → Microsoft → Windows → Windows Firewall With Advanced Security → Firewall

    Zusätzliche Tools (optional) Für detailliertes Logging und Visualisierung:

    • Wireshark: Echtzeit-Analyse des UDP-Verkehrs auf Port 51820
    • GlassWire: Netzwerk-Monitor mit grafischer Oberfläche
    • Sysmon + LogParser: Für tiefere Analyse und automatisierte Auswertung

    Fragestellungen

    • Was ist ein VPN und welches Hauptziel verfolgt dessen Einsatz?
    • Welche Unterschiede bestehen zwischen einem Site-to-Site-VPN und einem Remote-Access-VPN?
    • Wie unterscheidet sich WireGuard konzeptionell von klassischen VPN-Protokollen wie OpenVPN oder IPsec?
    • Welche Vorteile bietet ein selbst gehosteter VPN gegenüber einem kommerziellen Anbieter?
    • Welche Kriterien sollte man bei der Auswahl eines VPN-Protokolls oder VPN-Dienstes beachten?
    • Welche Sicherheitsrisiken bestehen, wenn ein privater Schlüssel eines WireGuard-Peers kompromittiert wird?
    • Wie kann man unter Windows den Netzwerkverkehr eines WireGuard-Tunnels überwachen oder protokollieren?
    • Welche Risiken bestehen bei der Nutzung eines mobilen VPN-Clients?
