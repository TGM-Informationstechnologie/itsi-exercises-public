    Rechnersicherheit

    Einführung

    Rechnersicherheit, oder Computersicherheit, umfasst alle Maßnahmen und Praktiken, die darauf abzielen, die Integrität, Vertraulichkeit und Verfügbarkeit von Daten und Systemen zu gewährleisten. Es beinhaltet den Schutz vor unbefugtem Zugriff, Datenverlust, Schadsoftware und anderen Bedrohungen.

    Ziele

    • Den eigenen Rechner untersuchen mit einer Kombination aus integrierten Tools, Microsoft-Utilities und Drittanbieter-Software.

    Kompetenzzuordnung

    Grundlagen der IT Sicherheit

    • gesicherte Informationsquellen für aktuelle Sicherheitsbedrohungen finden und benutzen
    • Quellen für Datenspuren benennen und einschränken;

    Voraussetzungen

    • Sysinternals Suite installiert
    • Dotnet Core Framewortk installiert
    • Dotnet AspNet Framework installiert
    • Attack Surface Analyzer 2.0 installiert

    Detaillierte Aufgabenbeschreibung

    Eine umfassende Übung zur Rechnersicherheit auf deinem eigenen Windows 11 PC kann dir helfen, sowohl dein praktisches Wissen zu vertiefen als auch deinen Rechner besser abzusichern.

    Aufgabe Grundkompetenz:

    Teil 1: Integrierte Windows-Tools nutzen

    Systeminformationen

    • Öffne die Eingabeaufforderung (cmd) und führe aus: systeminfo
    • Notiere dir Betriebssystemversion, Patch-Level, BIOS-Version, etc. Bist du aktuell?

    Windows-Sicherheitscenter

    • Öffne: Einstellungen > Datenschutz & Sicherheit > Windows-Sicherheit

    • Überprüfe:

      • Virenschutz (Microsoft Defender)
      • Firewall & Netzwerkschutz
      • App- & Browsersteuerung
      • Gerätesicherheit

    Microsoft Defender Offline-Scan

    • Starte einen Offline-Scan, um Rootkits und versteckte Malware zu erkennen:

      • Windows-Sicherheit > Viren- & Bedrohungsschutz > Scanoptionen > Microsoft Defender Offline-Scan

    Microsoft Update

    • Öffne Einstellungen > Windows Update und stelle sicher, dass dein System aktuell ist.
    • Alternative in Powershell: Get-WindowsUpdateLog

    Teil 2: Microsoft Attack Surface Analyzer

    Ein Tool von Microsoft, das Änderungen an der Angriffsfläche deines Systems erkennt.

    • Download: Attack Surface Analyzer (ASA_win_2.3.xxx.zip unter Assets)
    • Starte in der Eingabeaufforderung (wichtig als Administrator -- Start - Eingabeaufforderung - Als Administrator ausführen) mit "Asa gui"

    • Vorgehen (Empfehlung nutze den Guided Mode):

    • Baseline-Scan durchführen

    • Änderungen am System vornehmen (z. B. Firefox Software installieren)
    • Vergleichsscan durchführen
    • Unterschiede analysieren, danach Report anzeigen

    Hinweis: Nutze für RunID einen für dich wiererkennenden Namen, aktiviere alle Collecturen außer den FileSystem Collector.

    Teil 3: PowerShell-Sicherheitschecks

    Einige nützliche PowerShell-Kommandos: powershell

    Aufgabe Befehl
    Liste aller aktiven Benutzerkonten Get-LocalUser | Where-Object { \$_.Enabled -eq $true }
    Liste aller Administratoren Get-LocalGroupMember -Group "Administrators" (englisches Windows Installlation) oder Get-LocalGroupMember -Group "Administratoren" (deutsche Windows Installation)
    Status von Windows Defender Get-MpComputerStatus
    Liste aktiver Firewall-Regeln Get-NetFirewallRule | Where-Object { $_.Enabled -eq 'True' } | Format-Table Name, DisplayName, Direction, Action, Enabled

    Teil 4: Passwortmanager

    • Verwende ein Tool wie KeePass zur Passwortverwaltung und lege erste Passwörter an.
    • Teste deine eMail (Privat und TGM) mit Have I Been Pwned - Have I Been Pwned: Check if your email address has been exposed in a data breach.
    • Was spricht für die Verwendung eines Passwortmanagers, was dagegen?

    Teil 5: Tools und Logs analysieren

    Du bemerkst, dass dein Windows-System seit kurzem beim Start langsamer ist und der Lüfter ungewöhnlich laut läuft. Außerdem meldet dein Router erhöhte Datenübertragungen, obwohl du keine großen Downloads machst.

    • Wie kannst du die folgenden Tools in der Situationsanalyse verwenden?
    • Worauf achtest du, was sind Warnzeichen in den Tools für weitere Recherchen?
    • Welche nächsten Schritte kannst du setzen?

    Deine Analyse

    • Öffne eventvwr.msc und überprüfe:

      • Sicherheitsprotokolle (z. B. fehlgeschlagene Anmeldungen)
      • Systemprotokolle (z. B. Treiberprobleme)

    • Lade die Sysinternals Suite - Sysinternals | Microsoft Learn herunter.

    • Verschaffe dir einen Überblick über Tools wie:

      • Autoruns→ Autostart-Programme prüfen
      • Process Explorer → Prozesse analysieren
      • TCPView → Netzwerkverbindungen überwachen

    Aufgabe Erweiterte Kompetenz:

    Recherchiere zwei zentrale Sicherheitskonzepte, die in modernen Windows oder Linux Systemen eine wichtige Rolle spielen. Erkläre jedes Konzept in deinen eigenen Worten und beantworte dazu jeweils die folgenden Fragen:

    • Was ist das Konzept (Kurze Definition in eigenen Worten)?
    • Warum ist es sicherheitsrelevant (Welche Bedrohungen werden dadurch reduziert oder verhindert)?
    • Wo wird es konkret angewendet (Beispiel aus Windows, Netzwerken, Cloud, etc.)?
    • Welche Herausforderungen oder Grenzen gibt es (z. B. technische Einschränkungen, Benutzerverhalten)?

    Beispielhafte Konzepte zur Auswahl:

    • Trusted Platform Module
    • Trusted Boot
    • Security Policies
    • Encryption
    • Application isolation
    • ...

    Fragestellungen

    Im Rahmen dieser Übung benantworte folgende Fragestellungen:

    • Was versteht man unter dem Prinzip der „geringsten Rechte“ (Least Privilege)
    • Warum ist es wichtig, regelmäßig Sicherheitsupdates zu installieren?
    • Welche Funktion erfüllt eine Firewall auf einem Windows-System?
    • Was ist der Unterschied zwischen einem lokalen Benutzerkonto und einem Microsoft-Konto unter Windows 11 in Bezug auf Sicherheit?
    • Welche Informationen liefert das Tool Get-MpComputerStatus in PowerShell?
    • Was ist der Zweck des Microsoft Attack Surface Analyzer?
    • Welche Risiken bestehen, wenn Administratorrechte für alltägliche Aufgaben verwendet werden?

    Abgabe

    Die durchgeführten Tätigkeiten und gewünschten Elemente müssen in einem Protokoll zusammengefasst werden.

    Die Fragestellungen sollen ebenfalls in diesem Dokument bearbeitet werden.

    Das Protokoll ist eingescannt als ein PDF File zusammen mit einem evt. vorhandenem Projektfile auf moodle abzugeben.
